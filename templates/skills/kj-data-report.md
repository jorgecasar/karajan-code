## Coder instructions

When generating data reports:

1. **Structure the report** — follow this format:
   - **Executive summary** — 2-3 sentences with the key takeaway
   - **Methodology** — data sources, time range, filters applied, tools used
   - **Findings** — organized by theme or priority, each with supporting data
   - **Recommendations** — actionable next steps based on the findings
2. **Include data tables** — present data in tables with clear headers, units, and formatting. Align numbers right, text left.
3. **Highlight anomalies and outliers** — flag data points that deviate significantly from the norm. Quantify the deviation (e.g., "3x above average", "lowest in 12 months").
4. **Quantify everything** — use percentages, counts, averages, medians, and trends. Avoid vague language like "significant increase" without numbers.
5. **Show comparisons** — include period-over-period comparisons (MoM, YoY) and benchmarks where available.
6. **Cite data sources** — reference the specific dataset, query, or file that produced each finding.
7. **Separate facts from interpretation** — clearly distinguish what the data shows from what you conclude.
8. **Include limitations** — note any data gaps, sample size issues, or caveats that affect conclusions.

## Reviewer checklist

When reviewing data reports, check for:

- [ ] **Conclusions match data** — every claim is supported by the data presented? No logical leaps?
- [ ] **No cherry-picking** — all relevant data included, not just data that supports a narrative?
- [ ] **Methodology is sound** — data sources are appropriate, filters are justified, time ranges are sensible?
- [ ] **Numbers add up** — percentages sum correctly, counts are consistent across sections?
- [ ] **Anomalies explained** — outliers are acknowledged and investigated, not ignored?
- [ ] **Actionable recommendations** — recommendations are specific, feasible, and tied to findings?
- [ ] **Clear formatting** — tables are readable, units are specified, dates are unambiguous?
- [ ] **Limitations disclosed** — caveats and data quality issues are transparently noted?

## Output

Deliver the report in the structured format above, with:
- All data tables properly formatted
- Key metrics highlighted
- Clear separation between data, analysis, and recommendations
