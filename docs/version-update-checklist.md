# Version Update Checklist

Checklist of files and locations to review and update when releasing a new version of Karajan Code.

## karajan-code repo

### Version bump
- [ ] `package.json` — `version` field (`npm version patch|minor|major --no-git-tag-version`)
- [ ] `package-lock.json` — auto-updated by npm version
- [ ] `CHANGELOG.md` — new `## [x.y.z] - YYYY-MM-DD` section with Added/Changed/Fixed

### READMEs (role count, pipeline diagram, features)
- [ ] `README.md` — pipeline ASCII diagram, role count ("N roles"), role table
- [ ] `docs/README.es.md` — same changes in Spanish

### Role documentation
- [ ] `docs/custom-roles.md` — "Available roles" table (add/remove roles, update descriptions)
- [ ] `templates/roles/` — new role templates if roles were added

### Architecture docs
- [ ] `docs/adr/` — new ADR if architectural decisions were made

## karajan-landing repo (`~/workspace/karajan-landing`)

### Homepage and index
- [ ] `docs/src/content/docs/index.mdx` (EN) — `description`, `tagline` (version + role count), feature cards (role lists)
- [ ] `docs/src/content/docs/es/index.mdx` (ES) — same in Spanish
- [ ] `public/index.html` — role count, role lists (static HTML, not auto-generated)

### Getting started
- [ ] `docs/src/content/docs/getting-started/introduction.md` (EN) — role count, role enumerations, carousel
- [ ] `docs/src/content/docs/es/getting-started/introduction.md` (ES) — same in Spanish

### Pipeline guide
- [ ] `docs/src/content/docs/guides/pipeline.md` (EN) — pipeline diagram, role table, role count
- [ ] `docs/src/content/docs/es/guides/pipeline.md` (ES) — same in Spanish
- [ ] `docs/src/content/docs/guides/flows.md` — Mermaid flow diagrams (if pipeline order changed)

### Architecture
- [ ] `docs/src/content/docs/architecture/overview.md` (EN) — directory tree comment (role count), system diagram
- [ ] `docs/src/content/docs/es/architecture/overview.md` (ES) — same in Spanish

### MCP tools
- [ ] `docs/src/content/docs/guides/mcp-server.mdx` (EN) — tool count, new tools
- [ ] `docs/src/content/docs/es/guides/mcp-server.mdx` (ES) — same in Spanish

## Quick search commands

Find stale role counts:
```bash
# In karajan-code
grep -rn "15 roles\|16 roles" --include="*.md" --include="*.mdx"

# In karajan-landing (source only, skip public/)
grep -rn "15 roles\|16 roles" docs/src/ --include="*.md" --include="*.mdx"
```

Find stale version references:
```bash
grep -rn "v1\.57\|v1\.58" --include="*.md" --include="*.mdx" --include="*.html"
```

## Notes

- `public/` in karajan-landing contains **generated HTML** — some files need manual update (`public/index.html` homepage) while most are regenerated by the Astro build.
- Always check both EN and ES versions of every file.
- The CHANGELOG in karajan-code is the source of truth for what changed.
- Role enumerations appear in many places — use grep to find them all.
